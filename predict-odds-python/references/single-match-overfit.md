# 单场数据过拟合问题

## 现象

当球队仅有 1 场真实赛果时，Poisson 模型会**完全被那 1 场结果主导**，
即使该场是明显的失常表现。

### 2026-06-21 实盘案例

西班牙（FIFA #2）首轮 0-0 战平 Cape Verde（FIFA #70+），xG 仅 0.54。
模型只用这 1 场数据 → 预测西班牙 vs 沙特仅 35% 胜率。
而 Bet365 隐含 86%，Predict.fun 隐含 89%。

| 数据源 | Spain 胜率 | 原因 |
|--------|-----------|------|
| Bet365 盘口 | 86% | 反映 FIFA 排名+实力预期 |
| 模型（1场真实数据）| 35% | 被 Cape Verde 0-0 主导 |

## 根因

Poisson λ 估算依赖历史 xG 平均值。仅 1 场比赛时：
- `xg_for_avg = 0.54`（对 Cape Verde 的失常数据）
- 无其他数据缓冲 → λ 被严重压低

## 应对策略

1. **窗口期标识**：当 `matches_used ≤ 2` 时，标注「数据不足，以盘口为准」
2. **混合估算**：`λ_final = 0.3 × data_λ + 0.7 × tier_λ`（少量数据时降低数据权重）
3. **底线规则**：当模型概率与 Bet365 隐含概率差距 > 30pp 时，以盘口为准
4. **等第二场**：每队踢完 2 场后，xG 平均值会自然回归合理水平

## 何时信任模型

| matches_used | 可信度 | 建议 |
|-------------|--------|------|
| 0 | 纯 tier 估算 | 仅供参考，误差 ±30pp |
| 1 | 过拟合危险 | **以盘口为准** |
| 2 | 初步稳定 | 差距 < 15pp 时可参考 |
| ≥3 | 较可靠 | 差距 < 10pp 时可信任 Edge |
