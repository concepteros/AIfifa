# 第三轮小组赛分析框架

> 当世界杯小组赛进入第三轮（每队已打2场），分析每场比赛前必须完成以下检查清单。

## 第一步：构建积分榜

从 ESPN API 拉取已完成赛果，按组计算：

```
积分 → 净胜球(GD) → 进球数(GF) → 胜负关系
```

使用该组所有已完成比赛的结果。

## 第二步：确定每队的出线状态

| 状态 | 条件 | 行为预测 |
|------|------|---------|
| ✅ 已出线 | 积分确保前两名（6分以上，或4分+GD优势） | **极高轮换风险** — 可能轮换5-7名主力 |
| ⚠️ 打平即出线 | 1分足够 | 保守打法，不冒进 |
| 🔴 必须赢 | 输或平即淘汰 | 全力进攻，冒高风险 |
| ❌ 已淘汰 | 数学上无法出线 | 无动力，可能放飞或练兵 |

## 第三步：识别关键第三轮模式

### 🏷️ 握手局（Handshake Draw）
**条件**：双方同分，且打平足够双双出线。

**特征**：
- 双方都是「打平即出线」
- 平赔被市场低估（因为市场假设双方会争胜）
- Over 通常不成立（慢节奏，默契球）

**投注策略**：Draw @高赔率 + Under 2.5

**案例**：Switzerland vs Canada（6/25 BJT）— 双方4分，打平=5分携手出线。Draw @3.10 被低估。

### 🏷️ 轮换风险局
**条件**：一方已出线（6分），另一方生死战。

**特征**：
- 已出线方可能轮换5-7名主力
- 轮换后真实实力远低于盘口隐含
- 盘口基于全主力定价 → 已出线方赔率被高估

**投注策略**：博另一方+AH，绝对避开已出线方ML

**案例**：Czechia vs Mexico（6/25 BJT）— Mexico 已6分出线，Aguirre 轮换概率极高。Mexico @2.00 定价全主力，真实胜率远低于50%。

### 🏷️ 死局（Dead Rubber）
**条件**：双方均已淘汰，或一方已淘汰。

**特征**：
- 比赛结果不影响出线
- 心态放松 → 可能大开大合（对攻）或完全沉闷（散步）
- 赔率难以反映真实概率

**投注策略**：倾向回避。如果必须下注，走 Over（死局防守松懈），不走 ML。

**案例**：Bosnia vs Qatar（6/25 BJT）— 双方均1分已淘汰。

### 🏷️ 生死战（Must-Win）
**条件**：一方必须赢，另一方只需平。

**特征**：
- 必须赢方全力进攻，后防漏洞大
- 打平方保守防守，反击偷鸡
- 赔率可能高估必须赢方的能力

**投注策略**：倾向 Under（必须赢方拼命但另一边死守）+ 走打平方 AH

**案例**：Scotland vs Brazil（6/25 BJT）— Scotland 必须赢，Brazil 只需平。Brazil 1.33 危险区。

## 第四步：风险评估矩阵

| 模式 | ML可靠性 | Over/Under | 爆冷风险 | 首选投注 |
|------|---------|-----------|---------|---------|
| 握手局 | 低 | Under | 中 | Draw |
| 轮换风险 | 极低（对已出线方） | Under | 极高 | 对手+AH |
| 死局 | 低 | Over倾向 | 中 | 回避或Over |
| 生死战 | 中 | Under | 中（对强队） | Under + 弱队AH |
| 正常（全未出线） | 正常 | 正常 | 低 | 框架判断 |

## 与赔率框架的交叉

第三轮动力需要与三级赔率框架交叉判断：

- **碾压级 + 轮换风险**：碾压方轮换 → ML无价值。走对手 AH 或跳过。
- **危险区 + 握手局**：平赔被低估。Draw 是首选。
- **灰色区 + 生死战**：不确定性极高。走 Under 最安全。

## 数据源

使用 ESPN API 的 `STATUS_FULL_TIME` 事件匹配队名，计算积分+GD。参考 `scripts/build_real_fbref.py` 的 ESPN 调用模式。
